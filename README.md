# Al Power Portfolio

Next.js portfolio site for Al Power, covering the homepage, My Story, Illustrations, and Contact pages.

## Runtime note

On this machine the site should be run with the bundled Codex Node runtime rather than the system `node`, otherwise Next.js can fail because of native binary and code-signing issues.

Bundled Node path:

```bash
/Users/alpower/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin/node
```

## Local development

Start the dev server:

```bash
/Users/alpower/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin/node node_modules/next/dist/bin/next dev
```

Build for production:

```bash
/Users/alpower/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin/node node_modules/next/dist/bin/next build --webpack
```

Start the production build:

```bash
/Users/alpower/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin/node node_modules/next/dist/bin/next start -p 3000
```

Run lint:

```bash
/Users/alpower/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin/node node_modules/eslint/bin/eslint.js .
```

## Environment variables

No required environment variables are needed for a local build.

Optional:

```bash
CONTACT_EMAIL=alpower242@gmail.com
```

If `CONTACT_EMAIL` is not set, the contact form and mail links default to `alpower242@gmail.com`.

## Notes

- `robots.txt`, `sitemap.xml`, and `manifest.webmanifest` are generated by the App Router metadata routes.
- Portfolio images are kept at source quality in `public/`, with responsive delivery handled by Next.js image optimization.
